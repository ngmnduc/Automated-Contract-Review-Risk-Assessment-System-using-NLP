# Automated Contract Review & Risk Assessment System

An end-to-end NLP pipeline that automatically classifies legal clauses and assesses risk levels in contracts. Built with Legal-BERT fine-tuned on the LEDGAR dataset, served via FastAPI, and visualized through a Streamlit frontend.

## Step 1 — Download & Prepare Data

Run the ETL pipeline to download LEDGAR from HuggingFace, filter 5 target labels, clean text, balance classes, and split into train/val/test sets.

```bash
python etl_pipeline.py
```

This generates `data/train.csv`, `data/val.csv`, `data/test.csv`.

**Target labels extracted:**
| Label | Description |
|---|---|
| Terminations | Contract termination conditions |
| Indemnifications | Liability and indemnification clauses |
| Confidentiality | Confidentiality and NDA obligations |
| Governing Laws | Jurisdiction and governing law |
| Assignments | Rights transfer and assignment restrictions |

> **Note:** First run requires internet access to download LEDGAR (~2GB) from HuggingFace. Dataset is filtered to max 1,500 samples per class before splitting 80/10/10.

---

## Step 2 — Fine-tune the Model

```bash
python train.py
```

Fine-tunes `nlpaueb/legal-bert-base-uncased` on the prepared dataset. Saved weights will be written to `models/saved_weights/`.

**Training configuration:**
| Parameter | Value | Reason |
|---|---|---|
| max_length | 256 | 95% of clauses are under 250 words |
| batch_size | 16 | Fits standard GPU memory |
| learning_rate | 2e-5 | Prevents catastrophic forgetting |
| num_epochs | 3 | Sufficient for convergence; avoids overfitting |
| optimizer | AdamW | Better weight decay decoupling for transformers |

> **Note:** Training on CPU is slow (~2-4 hours). GPU recommended (~15-30 minutes). Model checkpoint is saved at the epoch with the best validation F1-score.

## Risk Engine Logic

The `RiskChecker` module uses deterministic keyword matching after classification:

| Risk Level | Behavior |
|---|---|
| HIGH | Matches critical keywords (e.g., "without notice", "unlimited liability", "perpetual") |
| MEDIUM | Matches moderate keywords (e.g., "30 days notice", "prior written consent") |
| LOW | Default if no HIGH or MEDIUM keywords are found |

> **Known limitation:** RiskChecker does not understand negation. The phrase "does NOT allow termination without notice" will still trigger HIGH risk due to substring matching on "without notice".

## Known Limitations

- **Single-label classification:** Model is forced to select one label per clause. Compound clauses containing multiple intents (e.g., both Governing Laws and Assignments) will have one label dropped.
- **EDGAR domain only:** High accuracy reflects standardized boilerplate phrasing in SEC filings, not deep legal reasoning. Cross-domain generalization is not guaranteed.
- **Sentence splitting:** `split_clauses()` splits on newlines and periods, which may incorrectly fragment clauses containing abbreviations (e.g., "Inc.", "Corp.").
- **RiskChecker negation:** Static keyword matching cannot handle negated or paraphrased risk language.
